# 🛡️ Reporte de Cumplimiento: compliance-reviewer

## 🎯 RESUMEN DE SEGURIDAD
Se han detectado riesgos de seguridad críticos que requieren atención inmediata antes de un despliegue real fuera del entorno de desarrollo.

---

**❗ Hallazgo**: Exposición pública de datos sensibles de funcionarios internacionales
**🔓 Vulnerabilidad**: Data Exfiltration / Exposición de Datos
**📍 Ubicación**: `public/data.json.gz`
**⚠️ Severidad**: CRÍTICA
**📜 Estándar/Ley**: Ley 19.628 de Protección de la Vida Privada (Chile) / OWASP 
**🛠️ Mitigación**: ELIMINAR el archivo de la carpeta pública de inmediato. Estos datos deben servirse desde el backend bajo autenticación JWT.

---

**❗ Hallazgo**: JWT persistente en `localStorage` (XSS Vulnerability)
**🔓 Vulnerabilidad**: Session Hijacking via XSS
**📍 Ubicación**: `src/services/api.js` y `src/context/AuthContext.jsx`
**⚠️ Severidad**: MEDIA
**📜 Estándar/Ley**: OWASP Top 10 A07 (Identification and Authentication Failures)
**🛠️ Mitigación**: Considerar el uso de `httpOnly Cookies` para el token de acceso o reducir el `ACCESS_TOKEN_EXPIRE_MINUTES` en el backend para minimizar el tiempo de exposición del token.

---

**❗ Hallazgo**: Token JWT en parámetros de consulta (Query Params)
**🔓 Vulnerabilidad**: Insecure Transport / Leakage in Server Logs
**📍 Ubicación**: `src/services/api.js` L140 (`getPdfViewUrl`)
**💥 Escenario de Fallo**: La URL del PDF aparece en los logs del servidor y en el historial del navegador con el token JWT visible.
**⚠️ Severidad**: MEDIA
**📜 Estándar/Ley**: OWASP A02: Cryptographic Failures
**🛠️ Mitigación**: Implementar tókens de un solo uso (One-Time Tokens) específicamente para previsualizaciones de documentos, o usar el método `downloadPdfAsBlob` que mantiene el token en los headers del request.

---

**❗ Hallazgo**: Falta de auditoría de roles en el Frontend
**🔓 Vulnerabilidad**: Broken Access Control (UX side)
**📍 Ubicación**: `src/components/layout/Navbar.jsx`
**⚠️ Severidad**: BAJA (UX) / MEDIA (API)
**📜 Estándar/Ley**: OWASP Top 10 A01 (Broken Access Control)
**🛠️ Mitigación**: Aunque el backend valida el rol (admin/revisor), el frontend debería ocultar opciones de gestión administrativa a usuarios sin permisos `admin` para evitar frustración del usuario al recibir errores 403.
