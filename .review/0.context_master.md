# 📘 CONTEXTO MAESTRO: CONAF Gestión de Contratos

## 📋 METADATA
- **Fecha**: 2026-03-26
- **Stack**: React 18, Vite 5/6, Tailwind CSS, Axios, Lucide Icons.
- **Backend Environment**: FastAPI (Python), PostgreSQL, JWT OAuth2.
- **Browser Support**: Navegadores modernos (ES2020+), Optimized for Chrome/Edge.
- **Criticidad**: ALTA (Manejo de datos legales y remuneraciones).

## 🎯 OBJETIVOS DE NEGOCIO
Digitalizar el flujo de firma física de contratos de CONAF, permitiendo la generación de documentos basados en plantillas, el seguimiento de estados (Pendiente, Revisado, Impreso, Esperando Firma, Completado) y el almacenamiento seguro de copias escaneadas.

## 🏗️ ARQUITECTURA TÉCNICA
- **Patrón**: Arquitectura basada en Componentes (React) con Servicios desacoplados (`src/services/api.js`).
- **Flujo de Datos**: Unidireccional. El estado global se maneja mediante Context API (`AuthContext`, `DataContext`). Los formularios de contratos usan un hook personalizado (`useContractForm`).
- **Web Standards Compliance**: Alto. Uso de `localStorage` para tokens, `Blob URLs` para visor de PDF, y transiciones CSS nativas.
- **Restricciones de Entorno**: Despliegue en Railway con volumen persistente para archivos PDF. CORS abierto en producción.

## ⚠️ RESTRICCIONES Y ÁREAS SENSIBLES
- **Audit Logging**: El historial de eventos (`/history`) es fundamental para la validez legal.
- **Invalidación Automática**: Cambios en el perfil del empleado DEBEN invalidar los contratos en proceso.
- **PDF Storage**: Los archivos NO deben ser públicos; deben servirse vía API con token.
