# 🛠️ Reporte de Errores: error-watcher

## 🎯 RESUMEN DE ESTADO
Se han identificado fallos potenciales en tiempo de ejecución relacionados con la integración de la API v2.0 y la gestión de referencias en el JSX.

---

**❗ Problema**: Desajuste en la firma de función `contratosService.create`
**📍 Ubicación**: `src/components/contracts/ContractDrawer.jsx` L130
**💥 Escenario de Fallo**: 1. Abrir Drawer. 2. Llenar formulario. 3. Clic en "Generar Contrato". 
**⚠️ Impacto**: Crítico (El contrato no se crea en el backend porque el payload se envía como `empleadoId`).
**🛠️ Fix Sugerido**: 
```javascript
// Antes
const newContract = await contratosService.create(payload);
// Después (ajustado a la API v2.0)
const newContract = await contratosService.create(employee.id, formData.template_id || 'contrato_indefinido', payload.datos_formulario);
```

---

**❗ Problema**: Referencia circular o error de importación en `PhysicalSignatureWorkflow` (anteriormente resuelto, pero validando persistencia)
**📍 Ubicación**: `src/components/contracts/workflow/PhysicalSignatureWorkflow.jsx`
**💥 Escenario de Fallo**: Al intentar realizar una acción de transición, el handler fallaba por nombres de variables inconsistentes.
**⚠️ Impacto**: Alto.
**🛠️ Fix Sugerido**: Asegurar que `contratosService` esté importado correctamente y los nombres de las funciones coincidan con `api.js`.

---

**❗ Problema**: Acceso a `localStorage` sin validación de disponibilidad
**📍 Ubicación**: `src/context/AuthContext.jsx` y `src/services/api.js`
**💥 Escenario de Fallo**: Navegadores con cookies/storage deshabilitado o en modo incógnito restrictivo.
**⚠️ Impacto**: Bajo/Medio.
**🛠️ Fix Sugerido**: Envolver los accesos en `try/catch` o usar un wrapper de almacenamiento seguro.

---

**❗ Problema**: Falta de limpieza de `setTimeout` en `ContractsPage`
**📍 Ubicación**: `src/components/contracts/ContractsPage.jsx` L109
**💥 Escenario de Fallo**: Si el componente se desmonta mientras una búsqueda está en curso, el callback intentará actualizar el estado de un componente desmontado.
**⚠️ Impacto**: Bajo (Memory Leak / React Warning).
**🛠️ Fix Sugerido**: El `return () => clearTimeout(delayDebounceFn)` ya está presente, pero falta validación de `isMounted` si la promesa de `searchEmployees` tarda más que el timeout.

---

**❗ Problema**: Error de parseo en `AuthContext`
**📍 Ubicación**: `src/context/AuthContext.jsx` L22
**💥 Escenario de Fallo**: Si `localStorage.getItem('user')` contiene un string malformado o "undefined", `JSON.parse` lanzará una excepción que romperá la aplicación al arranque.
**⚠️ Impacto**: Medio (Bloqueo de inicio).
**🛠️ Fix Sugerido**: `setUser(savedUser ? JSON.parse(savedUser) : null);` envuelto en un bloque `try/catch`.
