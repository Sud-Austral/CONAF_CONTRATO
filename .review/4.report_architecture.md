# 🏗️ Reporte Arquitectónico: architecture-reviewer

## 🎯 RESUMEN ESTRATÉGICO
El sistema presenta una arquitectura React SPA funcional y moderna pero con acoplamiento elevado entre componentes del Drawer y lógica de negocio específica.

---

**🏗️ Aspecto**: `src/hooks/useContractForm.js` y `ContractDrawer.jsx`
**📐 Patrón Detectado**: Prop Drilling / Acoplamiento de Estado
**⚠️ Riesgo Arquitectónico**: Fragilidad. El `Drawer` gestiona el estado de creación y sincronización con el backend directamente, en lugar de delegar a un `ContractServiceContextManager`.
**📈 Recomendación**: Extraer la lógica de creación e historial a un `ContractProvider` que centralice los cambios de estado de contratos para que el `Drawer` solo sea una vista.

---

**🏗️ Aspecto**: `src/services/api.js` (Interceptor Model)
**📐 Patrón Detectado**: Axios Interceptor Factory
**⚠️ Riesgo Arquitectónico**: Acoplamiento global con `window.location`.
**📈 Recomendación**: El interceptor de 401 que redirige al `/login` causa un hard-reload. Sería preferible emitir un evento que atrape el `App.jsx` para cambiar el estado de `isAuthenticated` y dejar que React Router maneje la navegación suave sin pérdida de estado.

---

**🏗️ Aspecto**: Generación de Documento en el Frontend (jsPDF)
**📐 Patrón Detectado**: Client-Side Resource Consumption
**⚠️ Riesgo Arquitectónico**: Carga de ejecución excesiva en browsers de gama baja (dispositivos móviles).
**📈 Recomendación**: Como se ha migrado a un Backend en Python (FastAPI), la generación de PDFs debería delegarse al servidor (server-side generation) y el cliente actuar solo como visor/descargador (`Blob URL`). (Nota: La v2.0 de la API ya contempla esto; la lógica antigua de `jsPDF` en el frontend debería ser eliminada paulatinamente).

---

**🏗️ Aspecto**: Sincronización de Contextos (`Auth` vs `Data`)
**📐 Patrón Detectado**: Dependency Direction Incorrecta
**⚠️ Riesgo Arquitectónico**: `DataContext` asume que `AuthContext` ya tiene el token persistido al ejecutarse, pero no hay una guarda explícita más allá de la protección de rutas.
**📈 Recomendación**: `AuthContext` debería ser el padre y `DataContext` hijo, pero con un renderizado condicional del hijo solo cuando `isAuthenticated` es `true`.
